# DXF 圖層解析與標線特徵萃取專案

## 前言

本專案主要使用 Python 進行 DXF 圖檔的讀取、圖層過濾與空間資料萃取，核心依賴套件為 `dxfgrabber`。

**dxfgrabber 的主要功能與使用方式**
`dxfgrabber` 是一個專注於「讀取」與「資訊抓取」的輕量級 DXF 解析套件。它的設計理念是將複雜的 DXF 幾何資料結構，轉化為 Python 開發者最熟悉的基礎資料型態（如 List 陣列與 Dictionary 字典）。使用者可以透過簡單的迴圈與物件屬性（如 `entity.layer` 或 `entity.dxftype`），快速遍歷整張圖紙、抓取特定圖層名稱、並讀取線段座標、多邊形、圖塊（Blocks）等幾何特徵，非常適合用於圖資的盤點與初步資料整理。

**dxfgrabber 與 ezdxf 的差別**
* **讀寫能力**：`ezdxf` 是一個全功能的強大套件，支援 DXF 檔案的讀取、建立、修改與儲存；而 `dxfgrabber` 為**純讀取 (Read-only)**，無法將修改後的物件存回原檔案。
* **學習曲線與維護狀態**：兩者其實出自同一位開發者之手，但作者目前已將維護重心完全轉移至 `ezdxf`。雖然 `dxfgrabber` 處於舊版維護狀態（不支援最新版的複雜修改特性），但其極簡的 API 結構使其在執行單純的「資料萃取與屬性讀取」任務時，語法依然比 `ezdxf` 更加直覺且易於上手。

## 安裝

在開始執行本專案前，請先確認您的開發環境已安裝 Python 3，並透過 pip 安裝 `dxfgrabber`：

```bash
pip install dxfgrabber
```

## 各程式介紹 (實務解析成果)

### `src/main.py`
專案的核心執行檔。負責載入 `data/road_markings.dxf` 檔案，並依序呼叫各個子模組來進行圖層過濾與特徵解析，最後統整輸出各標線物件的分析報告。目前已成功解析 AC1032 版本、包含超過兩萬個物件的圖資。

### `src/analyze_lines.py`
專門處理圖面上各類「標線」與「停止線」的讀取腳本。
* 精準過濾出 272 條獨立直線 (`LINE`)。
* 成功讀取並驗證了圖層的顏色屬性，除預設的 ByLayer (256) 外，亦抓取出被特別設定為顏色 8 的例外物件。

### `src/analyze_arrows.py`
針對「指向線」開發的特徵辨識與方向判斷模組（具備旋轉不變性）。
* **算法升級**：捨棄易受箭頭尾巴與圖面旋轉干擾的「中心點夾角算法」，改採**「頂點數量過濾 ＋ 向量外積 (Cross Product)」**的幾何特徵演算法。
* 成功從圖層中提取出 223 個聚合線 (`LWPOLYLINE`) 箭頭，並精準分類出真實道路現況：
  * 直行箭頭：23 個
  * 倒轉箭頭：25 個
  * 左轉箭頭：80 個
  * 右轉箭頭：22 個
  * 複合箭頭（如直行＋左右轉，頂點數 $\ge$ 14）：73 個

### `src/analyze_blocks.py`
負責尋找與解析含有特定圖示標線的模組。
* 克服了 AutoCAD 圖層命名的實務陷阱（精確比對了包含隱藏空白字元的 `' 026機車停等區(白)'` 圖層）。
* 破解機車停等區的幾何構成：成功從中分離出 66 條聚合線框線 (`LWPOLYLINE`)，並精準定位出 28 個名稱為**「機車人」**的專屬圖塊 (`INSERT`) 及其插入座標。

### `src/analyze_grids.py`
負責處理複雜幾何與網狀線圖形的模組。
* 具備辨識繪圖者習慣的除錯能力：透過實務解析發現，該圖檔之網狀線 (`013網狀線(黃)`) 並未依循常規採用 `HATCH` (填充) 指令來構成實體。
* 成功抓取出實際構成網黃線的 71 條 `LWPOLYLINE` (聚合線) 邊框物件，展現對非標準圖資的處理彈性。
